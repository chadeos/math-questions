Problem Set 3 (5 questions — includes one coding problem)

No problems are reused from Sets 1 or 2. Topics touched: bias–variance, regularization, gradient descent (1D), hash tables & sorting, recursion (coding).

1) Bias–Variance via Learning Curves

You plot learning curves (train error and validation error versus training set size):

  • Both curves are high and close together, and decrease slowly as you add more data.

(a) Is this primarily high bias or high variance?
(b) Name two concrete actions to improve performance given this diagnosis.

Expected answer type: Short conceptual explanation.

2) Regularization under Multicollinearity

You have many features that are highly correlated with each other (strong multicollinearity). You care about keeping groups of correlated features rather than picking only one.

(a) Which penalty would you prefer: Ridge, Lasso, or Elastic Net? Briefly justify.
(b) What qualitative effect does Lasso often have when predictors are strongly correlated?

Expected answer type: Short conceptual explanation.

3) Gradient Descent on a 1D Quadratic

Consider f(x) = (x - 5)^2. Start from x₀ = 0 and use gradient descent with constant learning rate η = 0.2.

  1. (a) Write the update rule for xₖ in terms of xₖ (no calculus beyond taking the derivative).
  2. (b) Derive a closed-form expression for xₖ as a function of k.
  3. (c) Compute x₁, x₂, and x₃.

Expected answer type: Short derivations and numeric values.

4) Hash Tables & Sorting – Concepts

(a) In a hash table with separate chaining and a uniform hash function, what is the expected-time complexity of a lookup in terms of the load factor α = n/m? Also state the worst-case complexity.

(b) Among the following sorting algorithms you know — bubble, selection, insertion, shell, merge, quick — name one that is stable and one that is not stable.

Expected answer type: Big-O statements and brief identifications.

5) Recursion — Coding Problem

Write a recursive Python function is_palindrome(s) that returns True if a string s is a palindrome and False otherwise, ignoring case and non-alphanumeric characters.

# Examples
is_palindrome("Racecar")          # True
is_palindrome("A man, a plan, a canal: Panama!")  # True
is_palindrome("hello")            # False

Expected answer type: Python function definition written by you.
